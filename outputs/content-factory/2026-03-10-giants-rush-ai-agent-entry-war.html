腾讯 QClaw、WorkBuddy，阿里 CoPaw，字节 ArkClaw：大厂集体“养龙虾”，中小创业者最危险的不是卷模型，而是错过入口

过去几天，国内 AI Agent 圈最值得重视的变化，不是又多了几个新名字，而是腾讯、阿里、字节几乎在同一时间把 OpenClaw 这一类产品往前推了一大步。这不是普通的跟风，也不是大厂日常的 PR 动作。我看到的信号非常清楚：AI 产业的竞争，正在从“谁的模型更强”转向“谁先拿到用户入口、工作流入口和执行入口”。

先把事实摆出来。

根据 IT 之家 3 月 9 日转述的消息，腾讯内测的 QClaw 不是从零重写一套 Agent 框架，而是围绕 OpenClaw 做的一次产品化封装。它的核心形态是本地一键启动包：下载安装后，可以在本地电脑上更轻松地部署 OpenClaw；如果本机已经装过 OpenClaw，还可以直接一键关联。报道里还提到，QClaw 默认可关联 Kimi、MiniMax、GLM、DeepSeek 等模型，并支持一键直连微信、打通 QQ。

再看 WorkBuddy。根据腾讯新闻 2 月 6 日报道，WorkBuddy 在内测阶段就已经有超过 2000 名腾讯内部不同岗位员工参与使用，场景覆盖数据处理与分析、本地知识库、文案创作、海报生成、自动化办公。根据界面新闻 3 月 9 日快讯，正式上线后的 WorkBuddy 完全兼容 OpenClaw 的 skills，最快 1 分钟即可完成配置并连接企业微信，还能接入 QQ、飞书、钉钉等工具。腾讯这两步连起来看，意思已经非常明确：QClaw 抢部署入口，WorkBuddy 抢办公入口。

阿里的动作也很重。根据 2 月 28 日腾讯新闻转载的资料，阿里巴巴宣布开源桌面 Agent 工具 CoPaw，支持本地与云端一键部署，允许开发者接入本地模型、自定义 Skills、扩展消息通道。更关键的是，CoPaw 把 Prompt、Hooks、Tools、Memory 等组件做了解耦，还引入了心跳机制和定时任务调度，能把对话中的决策、偏好与任务信息写入长期记忆。

字节这边也没有缺席。根据 IT 之家 3 月 9 日报道，火山引擎正式上线 ArkClaw，定位是开箱即用的云上 SaaS 版 OpenClaw。它不只是提供网页使用，还深度适配飞书 OpenClaw 官方插件。再往前一天，根据 IT 之家 3 月 8 日转述，OpenClaw 的飞书官方插件测试版已经上线；在用户授权后，OpenClaw 可以直接以“你的”身份看文档找资料、核对日历看档期、理解群聊上下文；同一天，飞书还把免费版 API 调用额度从每月 1 万次提升到 100 万次。这不是一个小修小补，而是平台侧在主动为 Agent 时代铺路。

把这几条线索放在一起，你会发现一个非常残酷也非常清晰的现实：大厂抢的不是“聊天机器人”，而是 AI 时代的操作系统入口。

一、这场战争真正抢的，不是模型，而是三层入口

很多人还在用 2024 年的视角看 2026 年的竞争，以为核心问题还是“谁更聪明、谁回答得更像人”。这个判断已经过时了。模型能力当然重要，但对大厂来说，模型已经越来越像电力、带宽和云资源：重要，但很快会被规模化、平台化、商品化。

真正稀缺的是三种入口：用户入口、工作流入口、执行入口。QClaw 解决的是先装起来、连起来的问题；WorkBuddy 把手伸进了工作台和办公流；CoPaw 把产品做成可重组、可扩展的工作站；ArkClaw 则通过 SaaS 路线和飞书插件吃掉办公上下文。这四个动作，本质上都不是在卷聊天，而是在卷入口、集成和默认使用路径。

二、为什么大厂一入局，创业难度会突然上一个台阶

这也是很多中小创业者最容易误判的地方。表面上看，大厂入局好像只是让市场更热；实际上，它会瞬间改变客户的预期、渠道的规则和产品的比较基准。

1. 通用 Agent 会被迅速压扁

如果今天还有人准备做一个“通用 AI 助手”，宣传语是“帮你做各种办公和自动化任务”，那接下来会非常难受。因为腾讯、阿里、字节、钉钉各自都有天然优势：入口、账号体系、权限体系和默认分发位都在它们手里。创业公司如果只是做一个“功能类似但品牌更小”的通用工具，很容易被客户问住：既然平台已经有了，我为什么还要额外买你？

2. 获客成本会显著上升

过去 AI 创业者最大的成本是教育用户“什么是 Agent”；现在这部分教育会被大厂替你做掉，但用户心智也会被大厂占住。你再去讲“AI 帮你自动写周报、处理表格、整理客户跟进”，客户会自然拿平台默认能力来比。一旦市场进入这个阶段，创业公司要么花更高成本证明自己“不一样”，要么就被归类成“替代品”。

3. API 与通道不掌握在自己手里

平台入口被大厂拿住之后，创业公司最难受的一点是：最关键的连接并不属于你。今天能接，明天可能改权限；今天能深度读写，明天可能只剩只读。创业团队最怕的不是竞争，而是自己依赖的那条链路突然从护城河变成租来的桥。

4. 客户对安全、审计、稳定性的要求会被整体抬高

大厂产品一出现，企业客户的参照系立刻变了。他不会只看“你能不能做这个功能”，而会追问私有化、权限分级、审计回放、上下文追溯和多系统治理。这会把大量只有 Demo 的团队筛掉。

三、大厂入局后，中小创业者最大的挑战是什么

如果只用一句话概括，我会说：最大的挑战不是技术追不上，而是产品定位还停留在“做一个助手”，而大厂已经在抢“默认入口”。

具体展开，我看到至少有五个挑战：通用能力不再值钱、场景太浅无法建立替代成本、缺少数据与流程护城河、大厂拿走可信感、人才和时间窗口都在压缩。没有这些判断，产品很容易陷入“看起来很强，实际卖不动”的状态。

四、但机会并没有消失，反而更集中、更清晰了

我反而觉得，大厂入局之后，创业机会比很多人想象中更大，只是位置完全变了。

机会一：不要做总入口，去做最后一公里

大厂擅长做平台、做分发、做统一入口，但很难把每个行业的细工作流都吃透。真正的机会在于：做一个岗位、一个行业、一个流程的深执行闭环。

机会二：做跨平台的中立协调层

企业真实世界里，不会只用一家平台。企微、飞书、钉钉、邮箱、表格、CRM、ERP、内部系统长期并存。平台方天然只会优先优化自己的闭环，这就给了中立层巨大的机会：统一权限治理、统一记忆层、统一编排层、统一观测与审计。

机会三：做“结果产品”，不是“AI 产品”

创业公司不要卖“我们接了最强模型”，也不要卖“我们有 Agent”。客户关心的是周报是不是不用再手写、销售是不是少漏跟进、内容是不是更稳定产出、团队是不是少招几个人也能运转。一旦你卖的是结果，就不容易被归类成“另一个聊天机器人”。

机会四：产品化服务会越来越有市场

Agent 时代，很多客户买的不是纯 SaaS，而是标准化底座加半定制交付：流程梳理、系统接入、权限配置、Prompt 与 Memory 调优、持续复盘。这种“产品化服务”模式反而很适合小团队起步。

机会五：生态插件、技能包、模板层会起来

当平台足够强、入口足够大，围绕它的生态也会跟着起来。未来很可能出现的新机会包括行业技能包、模板市场、Prompt+Tool+SOP 组合包、角色化 Agent 套件、安全与审计插件、垂直知识库增强层。

五、如果我是中小创业者，接下来怎么打，以及未来 12 个月会发生什么

如果我是现在的中小创业者，我不会再做一个“万能 AI 助手”，也不会把主要精力放在“接更多模型、接更多工具”上。接下来最重要的判断只有四个：我到底服务哪个岗位、解决哪个结果、嵌进哪个入口、凭什么不被平台默认能力替代。

我的打法会很简单。第一，做一个岗位的高频闭环。一个岗位、一个流程、一个 KPI，先把价值做深。第二，做跨平台中立层，帮助企业在腾讯、阿里、字节、钉钉的多入口环境里维持统一治理。第三，做可私有化、可半私有化的交付，特别适合对数据敏感、又不想完全被平台锁死的客户。第四，做结果可衡量的产品化服务，用业务结果而不是模型参数成交。

往后看 12 个月，我判断这四件事会越来越明显。第一，Agent 会全面平台化。它不再只是独立 App，而会嵌入微信、QQ、飞书、钉钉、浏览器、文档、CRM、客服系统。第二，部署与连接能力会快速商品化。QClaw、CoPaw、ArkClaw 这种产品越多，安装和接通能力越不值钱。第三，真正的壁垒会转移到流程理解、例外处理、审计治理和行业 know-how。第四，“AI 助手”会变成标配，“AI 同事”才会成为真正的付费项。

所以我给这波新闻下的最终结论是：腾讯 QClaw、WorkBuddy，阿里 CoPaw，字节 ArkClaw，再加上飞书对 OpenClaw 插件和 API 能力的放量，不是在做几款相似产品，而是在争夺 AI 时代最关键的东西——默认入口。对中小创业者来说，最危险的不是大厂进来，而是还以为自己在和大厂竞争模型。真正的竞争单位已经变了：不是“谁更聪明”，而是“谁更靠近真实工作、真实数据、真实结果”。如果入口被大厂拿走，创业者就别去正面抢门；最聪明的做法，是在门后面找到那个平台做不深、但客户又愿意持续付费的房间。
