# 微信公众号API发布经验总结

**最后更新**: 2026-03-06

## 问题背景

在发布公众号文章时，遇到脚本无法自动读取凭据的问题。

## 解决方案

### 1. 凭据配置位置

- **正确位置**: `/root/.openclaw/workspace/skills/content-factory/.env`
- **凭据来源**: `/root/.openclaw/credentials/wechat.json`

### 2. 配置步骤

```bash
# 读取凭据
cat /root/.openclaw/credentials/wechat.json | jq -r 'to_entries | map("\(.key)=\(.value)") | join("\n")'

# 写入 .env 文件
# WECHAT_APP_ID=wxf9400829e3405317
# WECHAT_APP_SECRET=f122506bb959353211175841f8ed039d
```

### 3. 正确使用方式

**错误方式**:
```bash
# ❌ 在命令行临时设置环境变量
WECHAT_APP_ID=xxx python3 script.py
```

**正确方式**:
```bash
# ✅ 直接运行脚本（会自动读取 .env）
cd /root/.openclaw/workspace/skills/content-factory
python3 -X utf8 scripts/wechat_publish.py --html "path/to/article.html"
```

### 4. 发布命令

```bash
cd /root/.openclaw/workspace/skills/content-factory
python3 -X utf8 scripts/wechat_publish.py --html "/root/.openclaw/workspace/output/2026-03-06-xxx.html"
```

**注意**: 必须加 `-X utf8` 参数，否则中文字符会变成 `\uXXXX` 编码

### 5. API 权限限制

- ✅ 获取access token: 成功
- ✅ 上传封面图片: 成功
- ✅ 创建草稿(draft): 成功
- ❌ 提交预览(preview): API未授权（需要更高权限）

**结果**: 草稿会自动创建，需登录 https://mp.weixin.qq.com/ 手动发布

## 关键要点

1. **凭据写入 .env**: Content Factory Skill的脚本会自动读取 `skills/content-factory/.env` 文件
2. **使用 -X utf8**: Windows环境下必须加这个参数防止编码问题
3. **手动发布**: 由于API权限限制，草稿创建后需手动在后台发布

## 重大错误：价格数据编造问题

**时间**: 2026-03-06

**错误描述**:
在撰写GPT-5.4 vs Claude对比文章时，我编造了GPT-5.4的API价格数据。

**错误数据**: $30/M (输入) / $180/M (输出)
**正确数据**: $2.5/M (输入) / $15/M (输出) - 来自platform.openai.com

**错误原因**:
1. 没有第一时间从官方网站核实数据
2. 主观猜测价格数据
3. 写完文章后才去查证，而不是之前

**深刻反思**:
- ❌ 禁止在文章中编造任何数据
- ❌ 价格、参数、排名等信息必须来自官方或权威来源
- ✅ 必须先查证，再写作
- ✅ 如果不确定，明确标注"待核实"

**正确流程应该是**:
1. 确定主题
2. **先查证所有数据**（官网、权威报道）
3. 整理信息
4. 写作
5. 再次核实
